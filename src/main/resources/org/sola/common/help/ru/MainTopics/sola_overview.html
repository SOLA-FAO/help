Solutions for Open Land Administration (SOLA)

Начатый в июне 2010 года, Solutions for Open Land Administration (Решения для Открытого Управления Земельными ресурсами) является 3-х годичным проектом, финансируемым правительством Финляндии. Группа Землепользования NRC Продовольственной и сельскохозяйственной организации Объединенных Наций (ФАО) инициировала и в настоящее время использует SOLA в качестве инструмента, который позволит принести пользу в обеспечении гарантий землепользования для бедных. Благодаря разработке и использованию бесплатного программного обеспечения с открытыми кодами (FLOSS), SOLA стремится сделать компьютеризированные кадастровую и регистрационную систему более доступной и устойчивой в развивающихся странах.

Изначальная версия SOLA представляет наилучшую международную практику в области управления земельными ресурсами и основывается на модели Land Administration Domain Model (LADM), в настоящее время опубликованной как проект международного стандарта Международной организации по стандартизации (ISO) под номером ISO 19152.

Базовая версия программного обеспечения была завершена в феврале 2012 года и в марте 2012 начато пилотное внедрение SOLA на Самоа и в Непале. В мае 2012 начата адаптация SOLA в Гане.

Адаптация базовой версии SOLA необходима для приведения ее в соответствие с местными законами, существующими земельными записями и принятой практики каждой страны где осуществляется внедрение SOLA, а также для удовлетворения конкретных задач агентства по управлению земельными ресурсами, в котором будет использоваться SOLA для выполнения функций регистрации и кадастра. Для содействия процессу адаптации были наняты и обучены местные разработчики программного обеспечения, которые в последствии осуществляли адаптацию, внедрение и поддержку решения. Местные команды разработчиков, как ожидается, должны стать ключевыми членами сообщества пользователей и разработчиков SOLA.

SOLA Релиз Кандидат

SOLA Релиз Кандидат (v1.1) является обновлением релиза SOLA Customization (V1.0) и содержит более 180 исправлений и улучшений, многие из которых были выявлены в ходе пилотных адаптаций. Версия Релиз Кандидат включает в себя все функциональные возможности, необходимые для базовых кадастровых и регистрационных операций в типовом регистрационном офисе, включая в себя:
  • Объединение и разделение участков
  • Кадастровая коррекция / переопределение
  • Поддержка ортофото
  • Регистрация прав собственности и ограничений
  • Печать сертификатов (регистрация, право собственности, недвижимость)
  • Функции управления заявлениями включая запрос сведений, отказ, одобрение / регистрация изменений
  • Отчетность по исполнению заявления
  • Управление документами
  • Проверка внесенных изменений с использованием настраиваемых, автоматизированных бизнес-правил
  • История действий по заявлению и его услугам
  • Администрирование пользователей (через SOLA администрирование)
  • Настраиваемый контроль доступа на основе ролей и определение бизнес-правил (через SOLA администрирование)
SOLA Релиз Кандидат является отправной точкой для адаптации решения под конкретную страну перед его внедрением в промышленную эксплуатацию в регистрационных офисах. Хотя и данная версия прошла инспектирование и тестирование, SOLA будет продолжать совершенствоваться и улучшаться в 2012 году командой разработчиков ФАО в Риме. Была развернута комплексная система управления версиями, которая включает все три пилотных страны и позволяет сделать доступными все улучшения базовой версии SOLA для команд разработчиков этих стран.

По этой причине, команда разработчиков SOLA приглашает членов сообщества SOLA, в частности разработчиков, работающих в области управления земельными ресурсами и потенциальных пользователей SOLA, принять участие в тестировании и предоставлении своих комментариев о версии SOLA Релиз Кандидат. Ваши комментарии могут быть отправлены на адрес neil.pullar@fao.org и команда ответить на ваши замечания и предложения.

SOLA Релиз Кандидат доступна в трех видах:

  1. Приложение Релиз Кандидат Web Start для демонстрации функций SOLA Desktop и SOLA Admin. Эти приложения развертываются с использованием технологии Java Web Start. Пользователи должны иметь доступ в Интернет и локальную установку Java SE Runtime Environment 7 для запуска Web Start приложений.
  2. Набор разработчика, который включает инструкции по установке и настройке средств разработки используемых SOLA, исходные коды, резервную копию базы данных и вспомогательные конфигурационные файлы. Набор разработчика SOLA также содержит инструкции о том, как обновить более ранние релизы набора разработчика SOLA для использования версии Релиз Кандидата. Набор разработчика может быть загружен по следующей ссылке http://www.flossola.org/downloads
  3. Как исходные коды из Github (https://github.com/SOLA-FAO). Для использования исходных кодов вам будет необходимо зарегистрироваться на Github и установить Git. Справочные материалы Github содержат подробные инструкции по установке и настройке. Обратите внимание, что SOLA использует 10 репозиториев Git. Подробнее о том, как управлять эти репозитории см. файл README.txt в репозитории code. С репозиториями идет постоянная работа командой разработчиков SOLA. Чтобы получить исходный код для версии Релиз Кандидат, извлеките тег candidate-release или ветку release.
Приложение SOLA Релиз Кандидат Web Start Набор разработчика доступны для загрузки по адресу http://www.flossola.org/downloads . Документация поддержки развертывания выпуска также доступны http://www.flossola.org/documents/release-candidate.

Дорожная карта

Адаптация SOLA для Самоа и Непала началась в марте 2012 года и завершилась внедрением в ноябре 2012 года. Адаптация для Ганы началась в мае 2012 года и внедрение ожидалось в конце 2012 или начале 2013 года. После завершения пилотных внедрений, General Availability (GA) релиз SOLA будет опубликован вместе с сопровождающей документацией.
